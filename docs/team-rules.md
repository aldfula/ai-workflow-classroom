# Team Rules

## 저장소의 목적

이 저장소는 수업 전체 인원이 함께 쓰는 PR 실습장입니다.
실제 팀 프로젝트는 별도 팀 저장소에서 진행합니다.

## 처음 PR 규칙

- 처음 PR은 반드시 자기 파일만 수정합니다.
- 공통 README, 공통 규칙 문서는 강의자 지시가 있을 때만 수정합니다.
- AI 에이전트가 수정한 변경도 사람이 `git diff`로 확인합니다.

## 브랜치 이름

```text
{cohort}/{github-id}/{agent}/{task}
```

## 충돌이 생기는 경우

같은 파일을 여러 명이 동시에 수정하면 충돌이 날 수 있습니다.
처음에는 각자의 파일을 수정해 충돌을 피하고, 이후 선택 실습에서 팀 공통 파일을 함께 수정해 충돌과 리뷰를 경험합니다.

## 되돌리기

공유된 기록은 지우기보다 새 커밋으로 되돌립니다.

```bash
git revert <commit-hash>
```
