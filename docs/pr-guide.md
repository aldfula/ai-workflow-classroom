# PR Guide

PR은 AI가 만든 결과물을 그대로 제출하는 곳이 아닙니다.
사람이 의도를 확인하고, 팀원이 검토할 수 있는 문맥으로 정리하는 곳입니다.

## PR 제목

```text
[AI 워크플로우 2026-06] glen15 / Codex 실습 기록 추가
```

## PR 본문

```md
## 변경 요약
-

## 사용한 에이전트
-

## AI에게 맡긴 일
-

## 사람이 검토한 것
-

## 개인 LLM-Wiki에 남긴 기록
-

## 확인한 것
- [ ] 정해진 기수 폴더 안에서만 수정했다
- [ ] 내 파일 또는 팀에서 합의한 파일만 수정했다
- [ ] 민감정보가 없다
- [ ] 출처가 필요한 내용에는 링크를 남겼다
```

## 리뷰에서 볼 것

- 파일 위치가 수업 컨벤션과 맞는가
- PR 제목에 수업명, 참여자, 에이전트가 드러나는가
- 자동 생성 흔적이나 과한 표현이 없는가
- 민감정보가 없는가
- 다음 사람이 이어갈 수 있는 설명이 있는가
